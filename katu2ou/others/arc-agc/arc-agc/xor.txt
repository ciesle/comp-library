    - l * (l+1) * ... * r = (0 * 1 * ... * (l-1)) * (0 * 1 * ... * r) とできる! 

    - 1 * 2 * ... * n の値は，1ビット目はn%4の値から定まり，それ以外(kビット目)は
        nのkビット目が0なら0, 1ならn%2の値から定まる.

    - 数列Aの偶数個の元のxorで表される <-> A1^A2, ... , A1^An のxorで表される

    - 非負整数からなる集合Aに含まれる二要素のxorの最小値は，Aをソートした時の隣接する二要素の
    xorの最小値に等しい(a<b<c なら min(a xor b, b xor c) < a xor c より)

    - 非負整数からなる集合Aに含まれる二要素のxorの最大値は，

    - 非負整数からなる集合A,Bに含まれる二要素のxorの最小値 -> Aの各要素に対して，
    [Aの上からi桁]とBの要素との比較をすることで解ける -> Bの要素からTrie木を作ることで
    O(NL) (Lはbit数)で解ける (単にmapを使う方針ではO(NLlogN)もかかる)

    


-- [xorの基底] --
    - https://miscalc.hatenablog.com/entry/2024/03/21/235130 など参照

    - Sを非負整数の集合として，X(S)をSで(xorにより)生成される値全体からなる集合とする．
    また,B(S)をSをF_2行列としたときの基底の一つとする．

    - Sの部分集合のxorでxが作れる時，作り方の個数はxによらない

    - chminでxor基底を求めるアルゴリズム
        vector<int> basis;
        for(int e: a){
            for(int b: basis)
                chmin(e,e^b);
            if(e)
                basis.push_back(e);
        }
    (今まで選んだ全ての基底の元でxorして小さい方をとったとき，0になっていなければ一次独立)
        -> この方法で，立っている最高位bit(msb)がdistinctな基底が得られる(つまり，行を
        降順ソートすれば階段行列になる)

    - 上のアルゴリズムを改良すると，msbが他の要素で立っていないような基底が得られる
    (簡約行列になる) 以下簡約基底と呼ぶ
        -> 基底を昇順にb_1,...,b_sとすると，b_1,...,b_sをxorに含めるかk_i={0,1}
        で表現した時，s桁の2進表記k_s...k_1で表される値kに対し，b_iらのxorで
        得られる値は，Sで作られる整数のうち小さい方からk番目の値である．
    
    - 奇数個選んで作れるもの: 今あるどのbitよりも上の余っているbitを1つ選び，全て
    の値で立てておく．そのbitが0なら偶数個，1なら奇数個選んだと言える．Sで生成できる
    全ての値を昇順に並べた時，最初の半分はそのbitが0に，残り半分はそのbitが1である．

    以上より，Sが与えられた時，次のような値が求められる．
        - xorを作る方法の個数
        - 作れる値のうち，あるbitが立っているものの個数
        - 作れるxorのうちk番目に小さいもの
        - 作れるxorのうちv以下で最大のもの(簡約基底上で貪欲的にいける)
        - 作れるxorのうち，xとのxorがk番目に小さいもの
        - 作れるxorのうち，xとのxorがv以下で最大のもの


-- [xor と mex] --
    - a xor b = mex{a' xor b, a xor b'|a' < a, b' < b}
