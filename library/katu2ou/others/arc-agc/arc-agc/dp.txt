-- [区間DP] --
    - 区間[l,r]の問題を，[l,k-1],[k+1,r]の問題に分解する典型方針(ex: ARC168D,ARC183C)

    - 数列nに対して，m個の区間[l,r]に対して満たすべき条件が与えられている時，
        dp[l][r] = (数列の[l,r]の部分を埋める方法のうち，[l,r]に含まれる区間の条件を満たすものの個数)
    を考える．[l,r]上の点kにおいて部分区間[l_i,r_i]に対する条件を満たすなら，[l,r]に含まれる区間であって
    kをまたぐものに関する条件を気にしなくて良くなるので，
        dp[l][r] += dp[l][k-1] * dp[k+1][r] * COM(r-l,k-l)
    とできる．条件を満たすk \in [l,r] に対してこれを行えばよい．

        -> この場合の計算量はふつうO(n^3)かかる

-- [実家(インライン)DP] --
    - 遷移をsegtreeを用いて高速化するやつ

    - NMマスのDPテーブルで解けるがNMでは大きすぎるような場合
        -> dp[i+1]がdp[i]と比べて定数個しか違わないような場合は，一次元dp配列を
        使いまわして，i->i+1への遷移を十分短時間で行うことができることがある．
        - 配るDPを貰うDPの形に書き換えると，整理できる形になったりする

    - コストが|x_i - x_j|となるような問題で多そう(minで一括更新できる形になる)

    - 区間が与えられて，[l_i,r_i]の中の要素のmax(min,and,or,...?)=d_i を満たすような
    数列の個数の数え上げ
    ex) Educational 84-F / ARC101F / ABC262H?

        -> 「ある状態がどこまで続くのかを添字に持つ」

-- [木DP] --
    - 大きさn,mの部分森の情報がO(mn)でマージできる時，N点からなる根付き木全体の情報は
    O(N^2)時間で計算できる (二乗の木DPとかいったりする)
    ex) ARC130D, ARC157E

    <ARC130D> 
        - n頂点からなる木を二部グラフとして黒と白に塗り分け，各頂点に1以上n以下の異なる整数を
        一つずつ割り当てる．黒い頂点にはそれに隣接するどの白い頂点よりも大きい整数を割り当てるような
        割り当て方は何通りあるか(N~3000)

        -> 根を適当に決めて，dp[i][j] : iを頂点とする部分木への割り当て方で，iに割り当てられた
        整数が小さい方から数えてj番目であるものの個数
            (マージする時にどのようにマージするかで二項係数が現れるやつ)
