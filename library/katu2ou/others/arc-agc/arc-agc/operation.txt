        1次元nimのような見た目をしている
        - 「a_nをa_{n-1}にして遷移できる局面に後手必勝があれば元の盤面は先手必勝」
        「そうでないなら，a_n -> a_{n-1}+1 とできるならよい」
    
    - 具体的な先手の必勝法を考えたり，ゲーム木の一部を考えたりすると思いつきやすそう

    <ARC126E>

    n回この操作を行った時の値の変量の総和の最大値をスコアとした時，スコアのn->inftyでの

        少なくともx変化し，かつちょうどx変化させるようにできる，つまりDが達成可能な上限になっている！

-- [区間の塗りつぶし] --
    - とりあえず逆から見てみる
    
    <ARC168D>
    - n個のマスと与えられたm個の区間があり，区間を一つ選んでその区間を塗りつぶすことを繰り返す．
    塗りつぶされたマスが増えるような操作は最大何回行えるか(n<=500)

    -> 状態が変化する操作のたびに，新しく塗りつぶされたマスを一つ選んでa_iとする．記録されたマスの列が
    a_1,...,a_k であるような操作が可能かを考え，kの最大値を求めれば良い．
        - dp[L,R] = (区間[L,R]に収まるような操作群のみを考えた時，状態が変化する操作数の最大値)とする．
        - <<操作を逆から見て>>みると，a_{i+1}が選ばれた時は，以降はL_j <= a_i <= R_j なる
        区間[L_j,R_j]について考慮する必要がなくなる (逆から見る効果!!)
        -> 問題が区間[1,a_i-1]と[a_i,n] に分割される (dp[l][r]の値を部分問題として考えられる)
    
    - 与えられたm個の区間のうち，区間[l,r]に含まれるものの個数cnt[l][r]を知りたい
        -> cnt[l][r]=cnt[l][r-1] + (左端がl以上で右端がr以下の区間の個数) を用いてO(n^3)で解ける


-- [区間の元の置換] --
    <ARC136B>
    - 長さnの整数列A,Bが与えられる．Aの連続する3元abcを選び，その部分をcabに並び替える
    操作を繰り返すことで，AをBに一致させられるか (N~5000, B_i~5000)

    -> 操作は，「ある元を選び，その元を右または左に2個移動させる」という操作と等価であり，
    このような操作(三元からなる巡回置換)は偶置換全体を生成する．よって，
    AとBの符号が等しいかで判定できる
        - Aに同じ値が2個以上存在する場合は，値の割り当て方を1組変えることを考えることで，
        A,Bの符号が等しくなるようにできる．よって一致させられる．
        - Aのどの二つの元も異なる場合は，符号が等しいかどうかで判定
