-- [頂点集合からの距離] --
    (頂点集合からの距離のmaxの最小化)
        <ARC116E>
            - N頂点の木が与えられる．うちK点を黒く塗り，木のどの頂点も黒い点との距離のminがT以下であるようにしたい．
            これが可能であるようなTの最小値を求めよ．(N~2e5)
                -> Tの判定問題に対してDPでいける．(O(NlogN)) 
                構築するなら適当に根付き木をとって，深い頂点からうまく埋めていくことでK個以下で
                距離Tの枠で覆えるかを判定する (O(N(logN)^2))

        <ARC108F>
            - N頂点の木の各頂点を白か黒かで塗り分ける．黒頂点間の距離のmaxをX,白頂点間の距離のmaxをY
            とするとき，max(X,Y)を2^N通りの塗り分け方について足した値を求めよ．(N~2e5)
                -> 木の直径を一つとり，直径の長さをLとおく．端点からの距離の
                maxがL,L-1,L-2,...であるような点に対して順に見ていき，どこで端点と
                同じ色の頂点が現れるかで値を計算する．
            
            性質: 頂点の部分集合Sをとるとき，max(Sの二点間の距離) <= max(直径の端点とSの点の距離)

-- [01 on Tree] --
    - 根付き木において，(親 -> 子) の順を保って頂点を並べる時のコスト(01ontreeでは転倒数)の最小化
    <AGC023F>
        - 頂点0を根とするn頂点の根付き木Gが与えられ，各頂点には01列としてs_i = 00...011...1 (0がk_i個,1がl_i個)
        が与えられている．n頂点を，各頂点について頂点がその親より後ろにくるように並べかえ，
        その順にs_iを結合した01列sを作る．sの転倒数の最小値を求めよ．

        -> 頂点iが持つ01列の0の個数をc0_i, 1の個数をc1_iとする．根でない頂点であって，
        c0_i/c1_iが最大となる頂点i=vを一つ選ぶ．sの最適解を実現する頂点の並びにおいて，
        親頂点p(v)とvは隣接しているとしてよい．(p(v)とvが離れていたら，vを近づけることで転倒数は
        増加しないから) よって，このような頂点についてp(v)とvをまとめることができ，
        転倒数の増加量 = (親の1の個数)*(子の0の個数) を加えた後に
        n-1頂点での問題に帰着される．あとはこれを繰り返し行えば良い．(c0_i/c1_iが最大である
        頂点を選ぶのにpriority_queueが，頂点をまとめた木を表現するのにunion-find treeが使える)

    <ABC376G> (派生系)
        - 頂点0を根とするn頂点の根付き木Gが与えられ，各頂点には重みa_i(a_i>=1は整数)が与えられている．
        n頂点を，各頂点について頂点がその親より後ろにくるように並べかえ，左からj番目の頂点をv_jとした
        時にj*a_{v_j}の総和を並べ方のコストとする．コストの最小値を求めよ．

        -> 重みa_iを01列0...01(0がa_i個)に置き換えると01 on Treeに帰着されることがわかる
        (a_iの寄与がj*a_{v_j} <-> s_iの0たちの左には1がj個ある
        <-> iはサイズjの親とまとめられる <-> iは数列のj番目にある(頂点の並べ方の制約(親->子)よりこれがいえる)) 

    - 一般に，「BはAの後に来ないといけない」という関係性がDAGを成すのであれば，
            「根付き木で親->子の順番を保つ」という条件に帰着できそう
