# Thistleの競プロ環境

## できること
- `link.sh`を動かすだけで、ほぼすべてのセットアップが完了する
- ショートカットを叩くとビルドや実行、ランダムチェックなどの機能が実行できる
- クリップボードの内容を入力として実行、というようなことができる
- `main.cpp`中でライブラリをインクルードしておくと、スクリプトで自動的に展開して提出できる形に整えてくれる。下のようなソースコードについて、コマンド一つでライブラリがすべて展開できる
```
#include<bits/stdc++.h>
using namespace std;
#include "library/include.h"
int main(){

}
```
- スニペットの登録を半自動的にやってくれる

## 使い方
- `include_what_you_use`をインストール
- `library/` にincludeしたいライブラリを入れます
- `snippet/` に入れたいsnippetを入れます
- `keybindings.json.bk`の中身を必要なものについて`keybindings.json`に入れます
- `link.sh` を実行します (`./link.sh comp` など、競プロ環境のディレクトリ名を引数にする)
- 環境が `../comp` に出来上がります

## 対応プラットフォーム
- Linux(Debian系)
- WSL(Debian系)

Macで動かしたい場合はLLMに頼んでください

## 各種コマンド
- `Ctrl+Shift+B`: 表示しているソースコードをビルドする（実行ファイルを手動実行したい場合、これをしないと見えるところに出てこない）
- `Ctrl+f5`: `main.cpp`が古い場合ビルドし、実行する
- `f5`: `main.cpp`が古い場合ビルドし、デバッガを起動する
- `Ctrl+f6`: `main.cpp`が古い場合ビルドし、`in.txt`を入力として実行する
- `Ctrl+Shift+f5`: `main.cpp`が古い場合ビルドし、`in.txt`にクリップボードの中身をコピーした上で、`in.txt`を入力として実行する
- `Ctrl+Alt+C` or `make copy`: `main.cpp`の内容を、インクルードしているライブラリを展開した上でクリップボードにコピーする（generic_copy）。
- `Ctrl+Alt+V`: クリップボードの中身を`in.txt`に貼り付ける
- `Ctrl+Shift+T`: ランダムテストを実行する
- `make snippet`: `snippet/`にあるファイル群をスニペットとして登録する

## デバッガの使い方
`f5`を押すと自動で起動する。デバッガを起動する前にデバッガウィンドウから起動するタスクを`Debug with input`に変更しておくことで、入力を`in.txt`から読むようになる

## ランダムチェックの使い方
`brute.cpp`にブルートフォースの愚直解を実装し、`gen.cpp`にテストケース出力コードを実装する。

`Ctrl+Shift+T`で出現するウィンドウに引数（`-n num`で実行回数、その他の引数は`gen`に渡される）を入力して実行することで、ランダムチェックが実行される。

`brute`と`main`の出力が食い違ったとき、入出力は`rnd_*.txt`に保存される。

## ライブラリインクルードの仕組みについて
`main.cpp`において、`library`以下のファイルをインクルードすると、generic_copyが`expand_include.py`を起動し、コード中のインクルードの依存関係を再帰的に解決し、展開したコードをクリップボードに保存する。
